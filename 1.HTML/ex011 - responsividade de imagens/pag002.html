Reproduzindo áudios

Para voltar para a página anterior, clique aqui.

Arquivo

Deve ser um arquivo .mp3, WAV ou OGG de preferência. Não deve ser nomeado com espaços, letras maíusculas ou acentuação.

Tags

Você pode utilizar a seguinte tag:

<audio src="CAMINHO DO ARQUIVO DE AUDIO"></audio>

Utilizando os parametros autoplay, para tocar automaticamente ou controls, para exibir o controlador de audio no seu site. Ficando assim:

<audio src="CAMINHO DO ARQUIVO DE AUDIO" autoplay controls></audio>

Porém utilizar essa tag dessa maneira tem um alto risco de incompatiblidade!

Correção:

Você insere a tag audio e remove o parametro src="#" que já vem como padrão. No lugar você insere manualmente a tag source:src, ficando dessa forma: 

<audio>
    <source src="CAMINHO DO ARQUIVO" type="descubra o mediatype no site iana">
</audio>

Você também pode inserir a tag P após os sources, então ela exibirá uma mensagem caso não consiga ler os arquivos de áudio.

A tag P não aparece, pois ela só será lida se o navegador NÃO TIVER NENHUM suporte à tag <áudio >. Ou seja, se o navegador não ler MP3, OGG ou outro qualquer, mas ler algum tipo de formato de áudio , então o resultado será o controle de áudio inativo (porque ele lê áudio, só não o formato passado), porém sem mostrar a mensagem do parágrafo com seu link para download. Para que o texto com link para download seja mostrado, o navegador não deve ter nenhum suporte a áudio, ou seja, não tem navegadores hoje sem esse suporte, todos tocam áudio, logo, a tag P nunca aparecerá.

Media types:

  • audio/mpeg
  • audio/ogg
  • audio/wav

Evite usar os arquivos .wav, pos são muito grandes.

Dentro da tag audio você pode inserior vários source:src, eles serão lidos na ordem em que os sources aparecem no código, e a leitura só continuará se não tiver conseguido ler o áudio atual, como uma espécie de busca.

Na tag audio, também existe o parametro preload="#", que recebe 3 valores: auto, metadata, none. O auto, só vai constar que o site terminou de carregar quando ele tiver carregado completamente o arquivo de audio, o que pode prejudicar a performance. O metadata só vai constar que o site está carregado quando ele carregar alguns dados essenciais do arquivo, como duração, tamanho, data, etc. E o none não vai esperar por nada.

Existe também o parametro loop, que irá reiniciar o áudio assim que ele terminar. 

<audio preload="auto" autoplay controls loop>
    <source src="midia/problemas-de-um-milionario-instrumentmp3" type="audio/mpeg">
    <source src="midia/problemas-de-um-milionario-instrumentmp3" type="audio/ogg">
    <source src="midia/problemas-de-um-milionario-instrumentmp3" type="audio/wav">
</audio>
